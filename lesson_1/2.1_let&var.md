# JavaScript `var` vs `let`

## Introduction

In JavaScript, `var` and `let` are both used to declare variables. Although they may look similar, they behave differently in terms of **scope**, **redeclaration**, **hoisting**, and **loop behavior**.

Modern JavaScript recommends using `let` (or `const`) instead of `var`.

# 1. Scope

## `var` (Function Scoped)

A variable declared with `var` is available throughout the entire function, even if it is declared inside an `if`, `for`, or `while` block.

### Example

```javascript
if (true) {
    var age = 21;
}

console.log(age);
```

### Output

```
21
```

---

## `let` (Block Scoped)

A variable declared with `let` only exists inside the block where it is declared.

### Example

```javascript
if (true) {
    let age = 21;
}

console.log(age);
```

### Output

```
ReferenceError: age is not defined
```

---

# 2. Redeclaration

## Using `var`

```javascript
var name = "Ali";
var name = "Ahmed";

console.log(name);
```

### Output

```
Ahmed
```

`var` allows redeclaring the same variable.

---

## Using `let`

```javascript
let name = "Ali";
let name = "Ahmed";
```

### Output

```
SyntaxError
```

`let` does not allow redeclaration within the same scope.

---

# 3. Reassignment

Both `var` and `let` allow changing the value.

## Example

```javascript
let age = 20;

age = 25;

console.log(age);
```

### Output

```
25
```

---

# 4. Hoisting

## `var`

```javascript
console.log(name);

var name = "Talha";
```

JavaScript internally treats it like this:

```javascript
var name;

console.log(name);

name = "Talha";
```

### Output

```
undefined
```

---

## `let`

```javascript
console.log(name);

let name = "Talha";
```

### Output

```
ReferenceError
```

`let` is hoisted but cannot be accessed before its declaration because of the **Temporal Dead Zone (TDZ)**.

---

# 5. Loop Behavior (`var` vs `let`)

This is one of the biggest practical differences.

## Using `var`

```javascript
for (var i = 1; i <= 3; i++) {
    setTimeout(() => {
        console.log(i);
    }, 1000);
}
```

### Expected Output

```
1
2
3
```

### Actual Output

```
4
4
4
```

### Why?

`setTimeout()` does **not** execute immediately.

Instead, it waits for **1 second**.

Meanwhile, the loop finishes executing.

The loop changes `i` like this:

```
i = 1
i = 2
i = 3
i = 4
```

After one second, all three `setTimeout()` callbacks execute.

Since `var` creates only **one shared variable**, every callback reads the final value of `i`, which is **4**.

### Visualization

```
Loop Starts

i = 1  --> Timer 1 Created
i = 2  --> Timer 2 Created
i = 3  --> Timer 3 Created

Loop Ends

i = 4

After 1 second

Timer 1 â†’ 4
Timer 2 â†’ 4
Timer 3 â†’ 4
```

---

## Using `let`

```javascript
for (let i = 1; i <= 3; i++) {
    setTimeout(() => {
        console.log(i);
    }, 1000);
}
```

### Output

```
1
2
3
```

### Why?

`let` creates a **new variable** during every loop iteration.

```
Iteration 1 â†’ i = 1

Iteration 2 â†’ i = 2

Iteration 3 â†’ i = 3
```

Each `setTimeout()` callback remembers its own separate variable.

After one second:

```
Timer 1 â†’ 1
Timer 2 â†’ 2
Timer 3 â†’ 3
```

# Real-Life Analogy

## `var`

Imagine there is only **one whiteboard**.

The teacher writes:

```
1
```

Then erases it and writes:

```
2
```

Then:

```
3
```

Finally:

```
4
```

Three students come later to read the whiteboard.

All of them see:

```
4
```

---

## `let`

Now imagine each student gets their own paper.

Student 1:

```
1
```

Student 2:

```
2
```

Student 3:

```
3
```

When they read their papers later, they still see their original values.

# When Should You Use Each?

## Use `let`

Use `let` when the variable value will change.

```javascript
let score = 0;

score += 10;
```

---

## Use `const`

Use `const` when the value should not be reassigned.

```javascript
const PI = 3.14159;
```

---

## Avoid `var`

`var` is mainly used in older JavaScript codebases.

For modern JavaScript development, prefer using `let` and `const`.

# Key Takeaways

- `var` is **function scoped**.
- `let` is **block scoped**.
- `var` allows redeclaration.
- `let` does not allow redeclaration.
- Both allow reassignment.
- `var` is hoisted with `undefined`.
- `let` is hoisted but stays in the **Temporal Dead Zone (TDZ)**.
- In loops, `var` creates one shared variable, while `let` creates a new variable for each iteration.
- Modern JavaScript best practice is to use `let` (or `const`) instead of `var`.
